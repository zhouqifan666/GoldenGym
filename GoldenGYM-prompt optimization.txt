# GoldenGYM 健身打卡系统 — 优化版提示词

-----

## 一、项目概述

**项目名称**：GoldenGYM 健身打卡系统（健身打卡助手）
**定位**：面向个人用户的运动打卡与目标管理 Web 应用，前后端分离架构，支持运动记录、统计分析、目标设定与日历可视化。

-----

## 二、技术栈

### 前端

- HTML5 + CSS3 + Vue 3（Composition API）+ Element Plus
- 路由：Vue Router 4
- 状态管理：Pinia（或 Vuex 4）
- HTTP 请求：Axios（统一封装，含拦截器）
- 构建工具：Vite

### 后端

- Java 1.8 + Spring Boot 2.x
- 架构模式：四层架构（Controller → Service 接口 → ServiceImpl → DAO/Repository）
- ORM：MyBatis-Plus（推荐）或 Spring Data JPA
- 数据库：MySQL 8.x
- 接口风格：RESTful API，统一响应体封装（code / message / data）
- 跨域处理：Spring 全局 CORS 配置
- 参数校验：javax.validation（@NotNull、@Min 等注解）

## 三、数据库设计

### 3.1 用户表 `user`

```sql
CREATE TABLE user (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password    VARCHAR(100) NOT NULL COMMENT '密码（建议BCrypt加密）',
    email       VARCHAR(100) COMMENT '邮箱',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间'
);
```

### 3.2 运动记录表 `exercise_record`

```sql
CREATE TABLE exercise_record (
    id             BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id        BIGINT NOT NULL COMMENT '关联用户ID',
    type           ENUM('有氧运动','力量训练','功能性训练') NOT NULL COMMENT '运动类型',
    date           DATE NOT NULL COMMENT '运动日期',
    duration       INT  NOT NULL COMMENT '运动时长（分钟）',
    calories       INT  COMMENT '消耗卡路里（kcal）',
    avg_heart_rate INT  COMMENT '平均心率（bpm）',
    note           VARCHAR(255) COMMENT '备注（可选）',
    create_time    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
);
```

### 3.3 目标表 `exercise_goal`

```sql
CREATE TABLE exercise_goal (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id         BIGINT NOT NULL COMMENT '关联用户ID',
    period          ENUM('每日','每周','每月') NOT NULL COMMENT '目标周期',
    target_days     INT  COMMENT '目标天数',
    target_duration INT  COMMENT '目标总时长（分钟）',
    target_calories INT  COMMENT '目标总卡路里',
    completed       BOOLEAN DEFAULT FALSE COMMENT '是否完成',
    create_time     TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
);
```

## 四、后端架构设计（四层）

### 4.1 统一响应体

```java
// Result.java
public class Result<T> {
    private Integer code;    // 200=成功，其他=失败
    private String  message;
    private T       data;
    // 静态工厂方法 success / fail
}
```

### 4.2 Controller 层

- `ExerciseController`：处理运动记录的 CRUD 请求及统计查询
- `GoalController`：处理目标的创建、列表、完成状态切换

```java
// 示例接口
@RestController
@RequestMapping("/api/exercise")
public class ExerciseController {
    @PostMapping("/record")          // 新增打卡记录
    @GetMapping("/records")          // 分页查询历史记录
    @GetMapping("/stats/week")       // 本周统计
    @GetMapping("/stats/month")      // 本月统计
    @GetMapping("/calendar/{year}/{month}") // 日历数据（返回有记录的日期列表）
    @GetMapping("/records/{date}")   // 查询某天的详细记录
    @DeleteMapping("/record/{id}")   // 删除记录
}

@RestController
@RequestMapping("/api/goal")
public class GoalController {
    @PostMapping("/create")          // 创建目标
    @GetMapping("/list")             // 目标列表
    @PutMapping("/{id}/complete")    // 切换完成状态
    @DeleteMapping("/{id}")          // 删除目标
}
```

### 4.3 Service 层接口

```java
public interface ExerciseService {
    Result<Void>       addRecord(ExerciseRecordDTO dto);
    Result<PageResult> getRecords(Long userId, Integer page, Integer size);
    Result<StatsVO>    getWeekStats(Long userId);
    Result<StatsVO>    getMonthStats(Long userId);
    Result<List<String>> getCalendarDates(Long userId, int year, int month);
    Result<List<ExerciseRecordVO>> getRecordsByDate(Long userId, String date);
}

public interface GoalService {
    Result<Void>           createGoal(GoalDTO dto);
    Result<List<GoalVO>>   listGoals(Long userId);
    Result<Void>           toggleComplete(Long id);
}
```

### 4.4 ServiceImpl 层

- 调用 DAO/Mapper 完成业务逻辑
- 统计接口需使用 SQL 聚合查询（SUM、COUNT、GROUP BY date）

### 4.5 DAO/Repository 层

- 使用 MyBatis-Plus 的 BaseMapper，复杂查询在 Mapper XML 中编写
- 典型 SQL 示例（日历查询）：

```sql
SELECT DISTINCT date 
FROM exercise_record 
WHERE user_id = #{userId} 
  AND YEAR(date) = #{year} 
  AND MONTH(date) = #{month}
```

## 七、关键 VO/DTO 定义

```java
// 打卡请求 DTO
public class ExerciseRecordDTO {
    @NotNull private Long userId;
    @NotNull private String type;     // 有氧运动 / 力量训练 / 功能性训练
    @NotNull private LocalDate date;
    @Min(1)  private Integer duration;
    private Integer calories;
    private Integer avgHeartRate;
    private String note;
}

// 统计响应 VO
public class StatsVO {
    private Integer count;       // 运动次数
    private Integer totalDuration; // 总时长（分钟）
    private Integer totalCalories; // 总卡路里
}

// 目标 DTO
public class GoalDTO {
    @NotNull private Long userId;
    @NotNull private String period;   // 每日 / 每周 / 每月
    private Integer targetDays;
    private Integer targetDuration;
    private Integer targetCalories;
}
```

## 八、交互细节与错误处理

1. **表单校验**：运动类型必选，日期必填，时长必填且 > 0；提交前前端校验，后端同步校验
1. **空状态**：所有列表页面在无数据时显示引导性空状态（图标 + 文字 + 操作按钮）
1. **Loading 状态**：所有 API 请求期间显示 Element Plus Loading 遮罩
1. **错误提示**：网络异常或后端返回非 200 时，使用 ElMessage.error 提示
1. **操作反馈**：打卡成功、目标创建成功均弹出 ElMessage.success；删除操作弹出 ElMessageBox.confirm 二次确认
1. **日历月份切换**：切换时重新请求当月有记录日期，清空右侧详情面板

## 九、扩展与注意事项

- 当前系统默认 userId = 1（单用户演示），后续可接入 JWT 登录体系
- 运动类型枚举若后续扩展，建议改为字典表而非 ENUM 字段
- 日历性能优化：前端缓存当前月已加载的打点数据，避免重复请求
- 移动端适配：导航栏在小屏下可折叠为汉堡菜单（可选）
